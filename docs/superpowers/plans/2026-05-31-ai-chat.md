## 目标

实现 AI 自由对话功能，支持：
1. **多会话管理**：创建、切换、删除会话
2. **上下文对话**：保持对话历史
3. **引用功能**：`@` 引用 Work/Material，将其内容注入到对话中
4. **流式响应**：实时显示 AI 回复（可选，v0.1 可以先用非流式）

enum AIChatRole {

  USER

}

## 功能详细设计

### Feature 1: 会话管理

**API**：
- `GET /api/ai/chat` - 获取会话列表
- `POST /api/ai/chat` - 创建新会话
- `GET /api/ai/chat/:id` - 获取会话详情（含消息历史）
- `DELETE /api/ai/chat/:id` - 删除会话
- `PATCH /api/ai/chat/:id` - 更新会话标题

**响应格式**：
```json
// GET /api/ai/chat
{
  "items": [
    {
      "id": "cm...",
      "title": "新对话",
      "createdAt": "2026-05-31T...",
      "updatedAt": "2026-05-31T...",
      "messageCount": 5
    }
  ]
}

// GET /api/ai/chat/:id
{
  "id": "cm...",
  "title": "选题讨论",
  "createdAt": "2026-05-31T...",
  "messages": [
    {
      "id": "cm...",
      "role": "USER",
      "content": "帮我分析一下这个作品",
      "attachments": { "workIds": ["cm..."] },
      "createdAt": "2026-05-31T..."
    },
    {
      "id": "cm...",
      "role": "ASSISTANT",
      "content": "好的，我看到你引用了...",
      "tokensUsed": { "input": 100, "output": 200 },
      "createdAt": "2026-05-31T..."
    }
  ]
}
```

### Feature 2: 发送消息

**API**：`POST /api/ai/chat/:id/messages`

**请求体**：
```json
{
  "content": "帮我分析一下这个作品",
  "attachments": {
    "workIds": ["cm..."],
    "materialIds": ["cm..."]
  }
}
```

**处理流程**：
1. 保存用户消息到数据库
2. 如果有 attachments，查询 Work/Material 内容
3. 构建完整的 Prompt（包含引用内容）
4. 调用 LLM
5. 保存 AI 回复到数据库
6. 返回 AI 回复

**Prompt 构建示例**：
```
[System] 你是一个短视频运营助手...

[User] 帮我分析一下这个作品

[引用的作品]
标题：xxx
描述：xxx
数据：播放 1000、点赞 50...

[Assistant] ...
```

### Feature 3: 引用功能

**UI 交互**：
1. 用户在输入框输入 `@`
2. 弹出搜索框，显示最近的 Work/Material
3. 选择后插入引用标记：`@作品标题` 或 `@素材标题`
4. 发送时，前端解析引用，提取 ID，放到 `attachments` 中

**简化版（v0.1）**：
- 不做 `@` 自动补全
- 提供"引用作品"和"引用素材"按钮
- 点击后弹出选择器，选择后添加到 attachments

---

### Feature 4: 自动生成会话标题

当会话有第一条消息后，自动用前 20 个字符作为标题：
```typescript
if (chat.title === '新对话' && chat.messages.length === 1) {
  const firstMessage = chat.messages[0].content;
  const newTitle = firstMessage.slice(0, 20) + (firstMessage.length > 20 ? '...' : '');
  await db.aIChat.update({
    where: { id: chat.id },
    data: { title: newTitle }
  });
}
```

## 技术细节

### 1. 上下文管理

每次发送消息时，需要包含历史消息作为上下文：
```typescript
const messages = await db.aIChatMessage.findMany({
  where: { chatId },
  orderBy: { createdAt: 'asc' },
  take: 20, // 最多保留最近 20 条消息
});

const chatMessages: ChatMessage[] = messages.map(m => ({
  role: m.role.toLowerCase() as 'system' | 'user' | 'assistant',
  content: m.content,
}));
```

### 2. Token 限制

为了避免超出 LLM 的 token 限制：
- 只保留最近 20 条消息
- 引用内容做截断（最多 1000 字符）
- 如果超出限制，提示用户"对话太长，请新建会话"

### 3. 引用内容格式化

```typescript
async function formatAttachments(attachments: { workIds?: string[], materialIds?: string[] }) {
  let text = '';
  
  if (attachments.workIds && attachments.workIds.length > 0) {
    const works = await db.work.findMany({
      where: { id: { in: attachments.workIds } },
      include: { metrics: { orderBy: { snapshotAt: 'desc' }, take: 1 } }
    });
    
    for (const work of works) {
      text += `\n\n[引用作品]\n`;
      text += `标题：${work.title}\n`;
      text += `描述：${work.description || '无'}\n`;
      if (work.metrics[0]) {
        text += `数据：${formatMetrics(work.metrics[0])}\n`;
      }
    }
  }
  
  if (attachments.materialIds && attachments.materialIds.length > 0) {
    const materials = await db.material.findMany({
      where: { id: { in: attachments.materialIds } }
    });
    
    for (const material of materials) {
      text += `\n\n[引用素材 - ${material.type}]\n`;
      text += `标题：${material.title}\n`;
      if (material.content) {
        text += `内容：${material.content.slice(0, 500)}...\n`;
      }
    }
  }
  
  return text;
}
```
